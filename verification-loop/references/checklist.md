# Verification Checklist

Use the smallest set of checks that proves the change.

## Typical order

1. Run the repo's fastest relevant tests first.
2. Run the feature-specific tests next.
3. Run typecheck or build checks when they matter to the slice.
4. Confirm the result maps back to the acceptance criteria.

## What to record

- Commands run
- Pass/fail result
- Any partial coverage gaps
- Follow-up work if verification is incomplete
